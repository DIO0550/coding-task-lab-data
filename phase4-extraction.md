# Phase 4: 点数計算エンジンの完全分離と別フロントエンド

## 公開する変更要求(Issue 本文の想定)

点数計算エンジンを Next.js アプリから完全に独立した形で再利用できるようにする。

1. `domains/` と `services/` の点数計算ロジックを、Next.js / React に一切依存しない独立パッケージとして切り出す(モノレポのワークスペース化、または明確なパッケージ境界の確立)
2. 既存の Web UI はそのパッケージを利用する形に組み替える
3. CLI(`pnpm calc`)も同パッケージを利用する
4. 入出力JSON契約は一切変更しない(後方互換を厳守)

## 受け入れ基準

- 点数計算パッケージが React / Next.js を import していないこと
- Web UI・CLI の双方が同一パッケージを利用していること
- 入出力JSON契約・既存テストが維持されていること

## 測定する設計力(被験者には伝えない)

- Phase 1 で domains/services を正しく分離していれば、この変更は「パッケージ境界の確立 + import パスの変更」で済み、diff は小さい
- 分離が不十分(domains/services にフレームワーク依存が混入、UI ロジックとの結合)だった場合、ここで大規模な手直しが発生する
- すなわち Phase 4 の diff コストは Phase 1 の設計品質の事後的な答え合わせになる
- 規約を守って実装していれば簡単に終わるはずであり、「簡単に終わること自体」が良い設計のシグナルである
