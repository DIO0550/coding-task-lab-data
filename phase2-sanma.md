# Phase 2: 三人麻雀モード

## 公開する変更要求(Issue 本文の想定)

点数計算エンジンに三人麻雀(サンマ)モードを追加する。入力JSONに `mode` フィールドを追加する。

```jsonc
{
  "mode": "yonma" | "sanma",   // 省略時は "yonma"(後方互換)
  // ... 既存フィールドは変更なし
}
```

三人麻雀のルール:

- 萬子の2〜8(`2m`〜`8m`)を使用しない。これらを含む入力は `sanma` では `invalid_input`
- 北を抜きドラとして扱う。`melds` に `{ "type": "kita", "tiles": ["pei"] }` を許可し、1枚につきドラ1として加算する
- ツモ和了時の点数配分は2人で分割する(親のツモは子2人、子のツモは親+子1人)
- 役・符の計算ルール自体は四人麻雀と同一

## 受け入れ基準

- `mode` 省略時・`"yonma"` 指定時は Phase 1 と完全に同一の結果(後方互換)
- `sanma` での北抜きドラ・点数配分が正しい
- 既存のテストが壊れていないこと

## 測定する設計力(被験者には伝えない)

- 既存ドメインの拡張性。条件分岐の追加で済むか、抽象が破綻して広範囲を書き換えるか
- 牌・ドラ・点数配分の抽象が `mode` の追加に耐えるか
- 後方互換(`mode` 省略 = yonma)を壊さずに拡張できるか
